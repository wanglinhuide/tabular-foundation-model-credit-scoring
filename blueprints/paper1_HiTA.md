## 2. 核心切入点 | Core Entry Point

### 致命盲区定义 | Fatal Blind Spot Definition

现有表格基础模型在面对以下极端约束时**全部失效**：

| 约束维度 | 现有方法 | 本文面对的现实 |
|---------|---------|--------------|
| 列名 | 假设部分重叠或可通过字符串匹配对齐 | **完全不相交** (col_A ∩ col_B = ∅) |
| 维度 | 假设 d_A = d_B 或通过填充对齐 | **d_A ≠ d_B**（如 d_A=30, d_B=15）|
| 目标样本 | 假设 N_B ≥ 1000 | **N_B ≤ 50** |
| 标注 | 假设目标域有完整标注 | **极少或无标注** |

**核心盲区**: 缺乏在**连续潜在语义空间**中进行列级"软对齐"的通用端到端机制，导致维度完全不匹配时现有方法要么崩溃，要么需要昂贵的人工特征工程介入。

## 3. 方法论与技术创新 | Methodology & Technical Innovation

### 系统总览

```
输入: T_A (源域表格, d_A列) + T_B (目标域表格, d_B列, 仅50条)
      ↓
[CSE] 列级语义嵌入器
  - 列名 → 预训练LM文本嵌入
  - 列内数值 → 统计特征(μ,σ,q25,q75) → 数值感知投影层
  - 输出: h^A_j ∈ ℝ^d (源域第j列语义向量)
         h^B_k ∈ ℝ^d (目标域第k列语义向量)
      ↓
[CSAN] 跨域软对齐网络 (可微分最优传输)
  - 计算传输代价: c(h^A_j, h^B_k) = 1 - cosine(h^A_j, h^B_k)
  - 求解 Π_{A→B} = argmin_Π Σ_{j,k} c(h^A_j, h^B_k)·Π_{jk}
  - 输出: 对齐矩阵 Π_{A→B} ∈ ℝ^{d_A × d_B}
      ↓
[HiTA Encoder] 异构感知表格Transformer
  - 通过 Π_{A→B} 将T_A表示投影至T_B的列空间
  - 交叉注意力: T_B中每行以对齐后的T_A作为Key/Value
  - 标准Transformer: T_B内样本间上下文建模
  - 输出: 违约概率 ŷ ∈ [0,1]
      ↓
[Meta-Learning Prior] MAML元先验蒸馏
  - 在多个源域上执行MAML外循环训练
  - 学习"快速适应新列结构"的初始化参数 θ*
  - 目标域仅需5-10步梯度更新即可收敛
```

### 核心公式

**目标函数**:
```
min_{θ} E_{A,B} [ L_CE(f_θ(T_B; Π_{A→B}(T_A)), y_B) + λ·W₂(P_A, P_B) ]
```

**最优传输对齐**:
```
Π_{A→B} = argmin_{Π ∈ U(μ_A, μ_B)} Σ_{j,k} c(h^A_j, h^B_k) · Π_{jk}
其中 U(μ_A, μ_B) = {Π ≥ 0 | Π·1 = μ_A, Π^T·1 = μ_B}
```

**MAML元学习**:
```
θ* = argmin_{θ} Σ_{(A,B)~p(T)} L_B(θ - α∇_θ L_A(θ))
```

### 四大技术创新点

1. **双通道列语义嵌入（CSE）**: 首次将"列名语义"与"列内数值分布"联合编码为统一列向量，突破了纯文本嵌入对数值含义理解的局限

2. **可微分最优传输软对齐（CSAN）**: 在语义隐空间中进行列间软对应，无需硬性One-to-One映射，支持Many-to-Many的跨域语义对应关系

3. **对齐先验驱动的交叉注意力（HiTA Encoder）**: 将对齐矩阵显式注入Transformer注意力计算，使目标域样本的推断过程能够参考源域语义相近的列信息

4. **跨域元学习信贷先验蒸馏**: 通过MAML在多个公开信贷数据集上蒸馏"快速列适应"元先验，确保50条样本下快速收敛

## 4. 实验设计 | Experimental Design

### 4.1 数据集构建

**源域数据集（12个，公开可用）**:
| 数据集 | 来源 | 样本量 | 列数 |
|--------|------|--------|------|
| Give Me Some Credit | Kaggle | 150,000 | 10 |
| Home Credit Default | Kaggle | 307,511 | 122 |
| Lending Club | Kaggle | 396,030 | 27 |
| German Credit | UCI | 1,000 | 20 |
| Taiwan Credit | UCI | 30,000 | 23 |
| PAKDD 2009 | Competition | 50,000 | 25 |
| ... (6 more) | Various | ... | ... |

**目标域数据集（3个，私有/半私有）**:
- 冷启动实验：每次仅开放 10/50/100/200 条样本
- 保证 col(target) ∩ col(any_source) = ∅

### 4.2 对比基线（7个）

| 方法 | 类型 | 对应痛点 |
|------|------|---------|
| XGBoost (source only) | 树模型 | 固定列依赖基线 |
| XGBoost (target fine-tune) | 树模型 | 小样本微调基线 |
| TabPFN v2 | ICL-TFM | 零样本ICL基线 |
| UniTabE | TFM | Schema无关基线 |
| CARTE | 图对齐TFM | 跨域对齐基线 |
| GTL (LLaMA-2) | 生成式TFM | 生成式预训练基线 |
| **HiTA (Ours)** | 最优传输+MAML | **本文方法** |

### 4.4 消融实验

| 消融配置 | 验证目标 |
|---------|---------|
| w/o CSE（随机列嵌入） | 验证双通道列嵌入的必要性 |
| w/o CSAN（硬对齐替代） | 验证软对齐vs.硬对齐的优势 |
| w/o Meta-Learning（直接微调） | 验证MAML元先验的快速适应贡献 |
| w/o Cross-Attention（直接拼接） | 验证对齐驱动注意力机制的作用 |

## 5. 审稿人视角的必中理由 | Reviewer Acceptance Arguments

### 对KDD审稿人
- ✅ **应用新颖性**: 信用评分冷启动是真实工业痛点，具有明确的经济价值
- ✅ **方法新颖性**: 最优传输+MAML在表格跨域场景的组合应用尚无先例
- ✅ **实验规模**: 12+3个数据集，7个基线，完整消融，超出KDD标准

### 对AAAI审稿人
- ✅ **理论严谨性**: 最优传输有Kantorovich理论背书；MAML有收敛理论保证
- ✅ **问题清晰性**: d_A≠d_B的数学形式化清晰界定了问题边界
- ✅ **贡献层次**: 理论（新问题定义）+ 方法（新框架）+ 实验（新Benchmark）三层贡献

### 潜在异议预案
- **异议**: "为何不直接用文本特征名称匹配？" → **回应**: 实验Section 4.1明确证明目标域列名为脱敏代码（x1, x2...），字符串匹配完全失效
- **异议**: "最优传输计算复杂度高？" → **回应**: 列数通常≤100，矩阵规模可控；附录提供Sinkhorn算法加速方案
- **异议**: "50条样本的实验设置是否realistic？" → **回应**: 附案例研究，展示真实银行新产品首月数据采集规模确实在此量级
