## 1. 论文题目 | Paper Title

**中文标题**: Tabular-RAG：面向冷启动信贷的检索增强决策智能框架——降低认知负荷、提升决策质量与增强监管合规性

## 2. 核心切入点 | Core Entry Point

### 管理科学视角的致命盲区

信息系统领域对AI辅助信贷决策的研究存在**三重系统性缺位**：

| 缺位维度 | 现有IS研究 | 本文填补的空白 |
|---------|-----------|--------------|
| **数据场景** | 聚焦"有历史数据/有标签"的成熟信贷产品 | **全新信贷产品冷启动期（零历史 + 极少样本）** |
| **知识单元** | RAG系统均为文本文档检索设计 | **纯结构化表格样本作为知识单元** |
| **管理价值** | 仅关注预测精度（AUC/KS） | **认知负荷降低 + 决策质量 + 监管合规三维评估** |

**核心研究问题**: 在全新信贷产品冷启动情境下，基于动态少样本表格检索增强的LLM决策支持系统能否同时实现：
1. 降低信贷经理的认知负荷（减少决策辅助时间和错误率）？
2. 提升信用评分精度与配给效率（经济价值量化）？
3. 满足金融监管对AI决策可解释性的合规要求？

## 3. 系统框架 | System Framework: Tabular-RAG

### 四层架构详解

```
┌─────────────────────────────────────────────────────────┐
│  Layer 4: 人机协同反馈闭环 (HACF)                         │
│  • 信贷经理审阅CoT推理链 → 标注纠错 → 实时回流HTKB         │
│  • 形成"检索→推理→审核→更新"闭环学习                       │
└────────────────────┬────────────────────────────────────┘
                     ↕ 审核结果 / 标注样本
┌────────────────────┴────────────────────────────────────┐
│  Layer 3: LLM-CoT 推理引擎                               │
│                                                          │
│  Prompt = [申请人特征(表格)] + [Top-K相似历史案例]          │
│           + [任务指令] + [CoT触发词]                       │
│                                                          │
│  五步CoT推理链:                                           │
│  Step 1: 风险因子识别（哪些特征是主要风险信号？）             │
│  Step 2: 历史案例参照（最相似的K个案例的结局如何？）          │
│  Step 3: 模式推断（当前申请人与高违约模式的相似度）           │
│  Step 4: 违约概率估计（0-100%范围 + 置信区间）              │
│  Step 5: 监管级解释（ECOA格式的拒贷/通过原因陈述）           │
└────────────────────┬────────────────────────────────────┘
                     ↑ 检索结果 (Top-K案例)
┌────────────────────┴────────────────────────────────────┐
│  Layer 2: 动态相似性检索引擎 (DSRE)                        │
│                                                          │
│  双通道检索:                                              │
│  通道A: 结构语义相似度 sim_struct(x_q, x_i)              │
│         = cosine(CSE(x_q), CSE(x_i)) ← 借用论文一CSE    │
│  通道B: 信贷风险模式相似度 sim_risk(x_q, x_i)            │
│         = RiskCluster(x_q) vs RiskCluster(x_i)          │
│  融合: sim_final = α·sim_struct + (1-α)·sim_risk         │
│  在线自适应: 随新样本积累动态更新权重α                       │
└────────────────────┬────────────────────────────────────┘
                     ↑ 查询向量
┌────────────────────┴────────────────────────────────────┐
│  Layer 1: 异构表格知识库 (HTKB)                           │
│                                                          │
│  构建流程:                                               │
│  源域A历史样本 → CSE列级语义嵌入 → 跨域统一向量索引          │
│  结构: {feature_vector, label, optional_annotation,      │
│         risk_cluster_id, domain_metadata}                │
│  索引: FAISS向量数据库（支持毫秒级Top-K检索）              │
└─────────────────────────────────────────────────────────┘
```

### CoT推理链示例（Prompt → Output）

**输入Prompt**:
```
申请人特征: {月收入: 4500元, 负债率: 78%, 工作年限: 0.5年, 
             历史逾期: 0次, 贷款用途: 消费}

最相似的3个历史案例:
案例1: {月收入: 4200, 负债率: 82%, ...} → 违约 (90天逾期)
案例2: {月收入: 5000, 负债率: 75%, ...} → 正常还款
案例3: {月收入: 4800, 负债率: 80%, ...} → 违约 (60天逾期)

任务: 请分析该申请人的信用风险，输出完整推理过程。
```

**CoT输出**:
```
【Step 1 风险因子识别】
主要风险信号: 负债率78%（超过行业警戒线70%），工作年限仅0.5年
（稳定性较低）。收入水平属中低档位。

【Step 2 历史案例参照】
3个最相似案例中，2个发生违约，违约率66.7%。负债率≥75%的2个
案例均出现60+天逾期。唯一正常还款案例的负债率（75%）低于本申请人（78%）。

【Step 3 模式推断】
本申请人与"高负债率+短工龄+中低收入"违约模式的匹配度约为82%。

【Step 4 违约概率估计】
预估12个月内违约概率: 71% (置信区间: 58%-84%)
风险等级: 高风险 (拒贷建议)

【Step 5 监管级解释（ECOA格式）】
主要拒绝原因:
1. 债务收入比过高（78%，超过本产品风险容忍阈值70%）
2. 工作稳定性不足（现职工作年限不足1年）
申请人可在以下方面改善后重新申请: (1)降低现有债务总额 (2)积累工作经历
```

## 4. 管理价值评估框架 | Management Value Assessment Framework

### 维度1：认知负荷降低（NASA-TLX量表）

**测量指标**（NASA-TLX六维度）:
- 脑力需求（Mental Demand）
- 体力需求（Physical Demand）
- 时间压力（Temporal Demand）
- 工作绩效（Performance）
- 努力程度（Effort）
- 挫折感（Frustration）

**实验设计**:
- 被试: 24名有信贷审批经验的从业者（工作年限≥2年）
- 组别: 纯人工决策 / 传统ML辅助 / Tabular-RAG辅助
- 任务: 在模拟冷启动环境中完成60个信贷申请审批
- 主假设H1: Tabular-RAG辅助组的NASA-TLX总负荷分值显著低于纯人工组（p < 0.05）

### 维度2：决策质量与经济价值

**冷启动期动态学习曲线**:
```
业务指标:
- 逾期率（30+天逾期占比）
- 信贷损失率（Loss Rate = 实际损失/放贷总额）
- 拒贷率（过度保守决策的代理指标）

时间点: 冷启动后 Day 7 / Day 30 / Day 60 / Day 90
```

**经济价值量化公式**:
```
ΔValue = Σ_t [ΔApprovalRate(t) × AvgLoanAmount × (1 - DefaultRate_RAG(t))]
          - Σ_t [DefaultRate_RAG(t) - DefaultRate_baseline(t)] × AvgLoss

其中 ΔApprovalRate = 因决策质量提升而额外批准的贷款比例
```

### 维度3：监管合规性（ECOA/GDPR评分）

**评分标准（5分李克特量表）**:
1. 拒绝原因的具体性（是否明确指出拒绝的具体因素）
2. 原因的可操作性（申请人是否知道如何改进）
3. 非歧视性（原因是否涉及受保护特征）
4. 逻辑一致性（解释与决策是否前后一致）
5. 监管可追溯性（能否满足监管部门的事后审查需求）

## 5. 实验设计 | Experimental Design

### 5.1 主实验：田野研究（Field Study）

**合作方**: 至少1家商业银行（匿名）
**场景**: 全新消费信贷产品上线初期（冷启动期90天）
**实验组**: 信贷经理使用Tabular-RAG辅助审批
**对照组**: 信贷经理按传统流程审批（无AI辅助）
**数据收集**: 每日审批数量、批准率、后续逾期追踪（12个月）

### 5.2 受控人机实验（Controlled Experiment）

**被试招募**: 24名从业者（随机分3×8组）
**环境**: 沙盒系统（模拟冷启动，历史案例池≤100条）
**测量工具**: NASA-TLX问卷 + 决策精度追踪 + 访谈记录

### 5.3 消融与敏感性分析

| 变量 | 测试范围 | 验证目标 |
|------|---------|---------|
| K（检索邻居数） | K ∈ {1, 3, 5, 10, 20} | 最优K值选择 |
| 检索策略 | struct-only / risk-only / hybrid | 双通道检索必要性 |
| CoT步骤数 | 3步 / 5步 / 7步 | 推理深度vs.效率权衡 |
| HTKB大小 | 100 / 500 / 1000 / 10000条 | 知识库规模影响 |
| LLM选择 | GPT-4o / Llama-3-70B / Qwen-72B | LLM骨干影响 |

### 5.4 扩展泛化测试

在4个非信用领域验证Tabular-RAG的通用性:
- 🏥 医疗保险核保（新险种冷启动）
- 📦 供应链风险评估（新供应商准入）
- 💻 P2P借贷平台（新用户画像冷启动）
- 🏢 小微企业信用评级（新行业进入）

- ✅ **政策影响**: 直接回应EU AI Act第22条（AI决策解释权）和中国《金融人工智能应用指引》，具有即时政策相关性
